Describing the applet breaks into two. Firstly before I can describe the param tags you need to understand the way items are defined.

All items in the applet are made up of an item string. This string is made up of all the arguments needed to define an item. Each argument is sperarted by a ';' semi colon. Each Item string is made up of 4 components

Title Path ; Icon ; Link ; Status string

Title Path

The title path gives the name of the item and its path separated by the forward slash or the backward slash. So the title path of one/two would break into folder one containing the item two. The title path one would be item one in the root directory. You may add items in any order if the directory for an item is not present the folder item is added for you. To add a directory you either add it indirectly by defining it in another item such as one/two defines the directory one. Or by defining item one and then upgrading it to a directory by adding a child item as it one/two. These paths may be of any depth say as in one/two/three/four we add the directories one two and three to add the item four.

Icon

The icon is a graphic that is displayed to the left of the item and is the file path to a ".gif" or ".jpg" image. Items may have two images separated by a "," comma eg folder.gif,open_folder.gif the second image being used if the item is a folder when its in an opened state.

Link

The link is a URL to a page or resource you wish to fetch when the item is activated. The URL need not be complete as the missing elements will be processed from the baseURL string which is a prama of the applet. This allows you to have links to local pages which you may not be able to give a full URL for. Eg with a baseURL = http://www.mcsoftware.com.au a link "local.html" would be added to the baseURL giving http://www.mcsoftware.com.au/local.html. Links may also define a frame to post the link into. This second argument is seperated by a "," comma eg "local.html,child" would post the page to the frame child. The default frame is _top which is the top most frame.

Status string

This is a string that appears after the URL link information on the status bar when an item is activated or the mouse is over the item. It is intended to give more information to the user as they browse the tree control.

When defining an item string components may be left blank or left off the end the only compulsory component is the title path.

Example

The item one/two

"One/two;http.gif;http://local.html;This is an item two under directory one"

may be defined without the end link of status

"One/two;http.gif;http"

Or again just as the title

"One/two"

Or without and icon or status text

"One/two;;http://local.html"

 

Rem out unused item strings

To rem out unused item strings or to allocate place holding items in numerical list you can now use a double * at the beginning of any item and that item will not be processed.

Example - With the list of tags below only item three would be processed and added to the list.

<param name=item1 value="**" >

<param name=item2 value="**Item Two;link" >

<param name=item3 value="Item Three;link" >

 

Prama Tags

The prama tags are used to add items to the control on intalisation. The

expanded

<param name="expanded" value="true">

Expanded tells the applet wether to initaly expand the root directory folder.

baseURL

<param name="baseURL" value="http://www.mcsoftware.com.au/">

This is the URL base used in links defined by diferent items. Items may define a complete URL but if they don’t missing items will be used from the baseURL. This defines where the local folder is from which links are referenced off.

rootTitle

<param name="rootTitle" value="Root Directory;folder.gif,o_folder.gif;;Status">

This is an item sting that allows you to define the root folder properties with a standard item sting (see item string above). You may define its title graphic, link and status text.

itemX

<param name="item1" value="One;folder.gif,o_folder.gif;;Folder One">

Items are added with the item tag. You may have as may items as you wish each with a sequential number added to the end eg item1, item2 item3, item4 .. itemX. Item numbers must be sequential and start from one as the program stops adding items if it can't find the next item. Item strings are a list of components and arguments that make up the item and tell the program how to display it. (See item strings above)

bgImage

<param name=bgImage value="sbox-title.jpg">

As promised I've added background images to the applet. The image's path is defined relative to the applets codebase and contains the filename of the image. 

Specifying colours

bgColor - set the background colour for the window (default white).

fgColor - This is the colour which is use to draw the layout links for the directory structure (default lightgray).

bgHighlight - The colour used as the background on highlighted text items (default rose).

textColor - The colour for text items( default black).

textHighlight - The colour of highlighted text (default white).

<param name=bgColor value="black">

<param name=bgHighlight value="lightgray">

<param name=fgColor value="A0A0A0">

<param name=textColor value="white">

<param name=textHighlight value="FF00A0">

Colours can be specified by a hex constant broken into the RGB components ("FFFFFF") or by a shorthand name ("white"). Available names are black, blue, cyan, darkgray, gray, green, lightgray, magenta, orange, pink, red, white, yellow or rose. If a colour is note specified or its invalid the control will use the default colour.

font

<param name=font value="Helvetica-plain-18">

The font used by the applet can be defined with a standard font string made up of the font name style and size. Styles can be either plain, italic, bold or bold italic. The size is the point size as a number and the font names can be any of the supported fonts for your platform. Widely supported standard fonts are Helvetica, TimesRoman, Courier and Dialog. The defualt font is Helvetica-plain-12 if the specified font is not available.

                offset

<param name="offset" value="25,5">

Use offset to set the coordinate offset of the top of the directory tree. This is useful if you want to shift the directory right and or down to fit better with a background graphic.

dotted

<param name="dotted" value=true>

Draw item links, the lines between items in dotted lines instead of solid lines.

showURL

<param name="showURL" value=false >

Stop the applet from showing the URL that it links to on the status bar. The links can be a bit messy also some people wish to hide exactly what is happening.

JavaScript

You can program the applet either using the prama tags on initialisation or by calling the public functions via JavaScript. From JavaScript you can add and remove items, highlight items, select the next and previous items and activate the current selection.

The publicly available functions are as follows;

Add an item to the control on the fly

public void addItem( String Item_string );

The item_string being the path and components of the item (see item string)

Highlight a particular item

public void highlight( String Item_string );

The item_string being the path of the item in question (see item string)

Highlight next visible item

public void next( );

Highlight previous visible item

public void previous( );

Activate current selection

public void activate( );

 

Example

add Item button adds an element to the control.

<input type="button" value="Add Item"

onclick="Tree.addItem('One/Three/Four;http.gif;http://www.mcsoftware.com.au,_top;Software')">

 

In this example we are using a standard html button

<input type="button" value="Add Item" >

with the acitvation command being

onclick="Tree.addItem('One/Three/Four;http.gif;http://www.mcsoftware.com.au,_top;Software')"

We are calling the function addItem with the argument being made up of an item string containing path and link information. (see item string)

<param name="item4" value="Item5;;;Item 5">

<param name="item5" value="Item6;;;Item 6">

</applet>
