... type:

    List(T)

  • A typeIde is well-formed
  • Wrong is well-formed
  • Nil is well-formed
  • A unification variable is well-formed, iff
    • its bound is well-formed
    • its instantiation is well-formed (always true ???)
  • A method type is well-formed, iff
    • its signatures are well-formed
    • its return type is well-formed
  • A method suite (aka object type) is well-formed iff
    • its Self signature bound is well-formed
    • all its method types are well-formed
  • A Var type is well-formed iff its parameter is well-formed
  • A type application is well-formed iff
    • the applyee (???) is an oper
    • the formal parameters comply to the signatures of the oper
  • A type operator is well-formed iff
    • its signatures are well-formed
    • its range is well-formed

... value:

    3 > 2 ? { "Yes" } : { "No" }

  • TL2AssertValue: check that value's type is subtype of Bool, result is Void
  • X_TL2AssignValue: check that value's type is subtype of the ide's type, result is that type.
  • X_TL2BlockValue: check all types, result is last value's type, Void if empty block
  • TL2Bool2Value ( And, Or, Xor ): check that both operands' types are subtype of Bool, result is Bool
  • X_TL2IdeValue: result is declared/inferred type
  • TL2IfValue: check that condition's type is subtype of Bool, result is Void
  • ?X_TL2Value: this is a send to Bool
  • X_TL2LetValue: Maybe introduce a UniVar, check that assigned value's type (if any) is subtype of the declared type, result is the declared type
  • X_TL2LiteralValue: result is the type of the literal
  • X_TL2SendValue: lookup method, check bindings vs method signatures, result is method's range
  • X_TL2SuperSendValue: lookup method in X_TL2CPL type, ...
  • TL2TryValue: check body, check handler, result is Void
  • LoopValue:
    while(x > 0) { result := result + a[x], --x, result } : Int ?
    check that condition's type is subtype of Bool, check body, result is Void ( or type of body ).

Implementation

  • there is a class for every type in the AST:
    • X_TL2IdeType ( identifiers as types : String )
    • X_TL2ApplyType ( appearant applications of type opers to type arguments : List(String)
    • X_TL2NilType
  • additional types:
    • X_TL2ObjectType (introduced by the X_TL2TypeChecker for every class)
    • X_TL2OperType (introduced by the X_TL2TypeChecker for every parameterized class)
    • X_TL2VoidType (a special, empty methodsuite)
    • X_TL2MethodType (allocated by the X_TL2ClassManager )
    • X_TL2SubstType (introduced by the X_TL2TypeChecker if the range of a substitution is to be constrained to a single type, see note below)
    • X_TL2UnifyType ( introduced by the X_TL2TypeChecker for type inference for omitted types at
      • let-bindings: let x = expr
      • method-calls: cond ? { ifTrue } : { ifFalse }
    • X_TL2CPLType ( by the compiler for types in the super list of classes )
