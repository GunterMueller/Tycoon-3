Installation von Jam/MR auf MS Windows NT/95/98

Vorarbeiten:

- Beschaffung aktuellen Version von Jam/MR, z.B. bei Perforce Software.
- Entpacken des Tar-Archives in ein Verzeichnis.
- Jam.patch in das Verzeichnis kopieren.
- Patch anwenden: patch -u -b <Jam.patch

Durch diesen Patch werden folgende Änderungen an den Quelltexten erzeugt:

- Generierung der nötigen Makefiles: Makefile.nt, Makefile.w95, Makefile.w98
- Windows NT: Die Definition von MAXLINE (Jam.h) wird auf 10K (10240) gesetzt
- Windows 95/98: Zusätzliche #ifdefs für die bedingte Compilierung
- Windows 95/98: Andere Semantik von spawnvp (in execdos.c)

Umgebungs-Variable auf Pfad des verwendeten C-Compilers setzten (siehe Jam-Readme), z.B.:

SET MSVCNT=F:\Lang\DevStudio\VC

für Microsoft Visual C++ 5.0. Vorsicht ist geboten, wenn der Pfad-Name Leerzeichen enthält, z.B.:

"C:\Program Files\DevStudio",

dies verträgt Jam nicht. Eine Lösung ist dann:

SUBST Y: "C:\Program Files\DevStudio"       und
SET MSVCNT=Y:\VC

Übersetzen:

Vor dem Übersetzen gegebenenfalls erst einmal VCVARS32.BAT ausführen (auf DevStudio\VC\Bin), um die Umgebungsvariablen für MS Visual C++ zu setzen. Erstmalige Übersetzung der Jam-Sourcen mit nmake, später kann Jam selbst benutzt werden. Je nach Plattform das entsprechende Makefile benutzen:

System Make-Prozeß Bin-Verzeichnis
Windows NT nmake -f makefile.nt bin.winntx86
Windows 95 nmake -f makefile.w95 bin.win95x86
Windows 98 nmake -f makefile.w98 bin.win98x86

jam.exe kann aus dem erzeugten Bin-Verzeichnis in ein Verzeichnis kopiert werden, daß sich im PATH befindet. Es sind keine weiteren Dateien außer jam.exe nötig.

Je nach dem, auf welcher Plattform man ist, und welche Dinge man schon installiert hat, benötigt man noch einige Tools (rm, touch), die ich hier mit auf dem Tools-Verzeichnis gelegt habe. Hier befindet sich auch diff und patch, falls man weitere Modifikationen an den Quelltexten vornimmt. Alle Tools stammen aus der Cygnus GNU-Tools package (beta 19): http://www.cygnus.com/misc/gnu-win32/
